The is the MODERNIZE branch

GitHub Writing Workshop Guide

Rewriting Walter Benjamin's "The Work of Art in the Age of Mechanical Reproduction"

Initial Setup

# Configure Git
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# Generate SSH key
ssh-keygen -t ed25519 -C "your.email@example.com"

# Clone repository
git clone https://github.com/douglasgoodwin/TWAAMR.git
cd TWAAMR

Project Structure

TWAAMR/
├── .gitignore           # Excludes temporary files
├── chapters/           
│   ├── 01.md            # Your assigned chapter
│   └── notes.md         # Research and drafting notes
└── style-guide.md       # Formatting standards

Branch Structure

main
└── student/<username>
    └── chapter1
        ├── draft
        └── review

Team Workflow

  1. Repository Setup
  2. Main branch is protected - only accepts reviewed pull requests
  3. One designated "Merge Master" handles all merges

  4. Each student works in their own namespace

  5. Create Your Space

    git checkout main
git pull
git checkout -b student/yourname
git checkout -b student/yourname/chapter1/draft

  6. Save Changes

    git add chapter1.md
git commit -m "draft: initial rewrite of introduction"

  7. Backup Work

    git push origin draft/chapter1

Writing Guidelines

  1. Format chapters in Markdown: ```markdown

    Chapter Title

Section Heading

Your rewritten text here. Include citations:

Original Benjamin: "Quote here" ```

  1. Commit standards:
  2. draft: Initial writing
  3. revise: Content changes
  4. style: Formatting only
  5. cite: Citation updates

Peer Review Process

  1. Request review:

    git push origin draft/chapter1
# Create pull request on GitHub

  2. Review changes:

  3. Comment on specific lines
  4. Suggest alternative interpretations

  5. Check style consistency

  6. Incorporate feedback:

    git checkout draft/chapter1
# Make changes
git commit -m "revise: address reviewer comments"

Submitting Changes

  1. Push your branch:

    git push origin student/yourname/chapter1/draft

  2. On GitHub:

  3. Create pull request
  4. Select Merge Master as reviewer
  5. Add description of changes

  6. Wait for review/approval

  7. After approval, Merge Master will:

  8. Review changes
  9. Handle merge conflicts
  10. Merge to main
  11. Delete merged branch

Final Submission

  1. Update from main:

    git checkout draft/chapter1
git pull origin main
git rebase main

  2. Final checks:

  3. [ ] Markdown formatting clean
  4. [ ] Citations verified
  5. [ ] Style guide followed

  6. [ ] Peer review addressed

  7. Merge to main:

    git checkout main
git merge draft/chapter1
git push origin main

Troubleshooting

  1. Resolve conflicts:

    git status
# Edit conflicted files
git add <resolved-files>
git commit

  2. Undo last commit:

    git reset --soft HEAD~1

  3. Recover lost work:

    git reflog
git checkout -b recovery HEAD@{1}

Writing Milestones

  1. First Draft
  2. Read original chapter
  3. Outline key concepts

  4. Write initial translation

  5. Revision

  6. Add modern examples
  7. Connect to adjacent chapters

  8. Format citations

  9. Final Polish

  10. Apply style guide
  11. Complete peer review
  12. Prepare for merge

Need help? Create an issue on GitHub or contact your instructor.
